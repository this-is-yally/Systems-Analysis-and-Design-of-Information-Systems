# Розділ 8. Обґрунтування вибору типу бази даних

## 8.1 Вимоги до даних

Система зберігає структуровані, взаємопов’язані дані (книги, користувачі, позики, бронювання). Необхідні:
- Відношення між сутностями (Foreign key)
- Транзакції для видачі/повернення (атомарність)
- Звіти та складні запити (JOIN, агрегації)

## 8.2 Підхід: реляційна СУБД

Вибір: **реляційна база даних** (PostgreSQL / MySQL / MariaDB).

Переваги:
- Схема з чіткими таблицями та залежностями.
- Підтримка транзакцій (ACID).
- Індекси для швидкого пошуку.
- Валідація через обмеження (NOT NULL, UNIQUE).

## 8.3 Альтернативи

- **NoSQL (MongoDB)**: добре для неструктурованих документів, але втрачається простота зв’язків між сутностями.
- **Graph DB**: корисно при складних графових запитах, але зайве для стандартної бібліотечної системи.

## 8.4 Висновок

Реляційна СУБД є найкращим вибором для даної системи через явність зв’язків, підтримку транзакцій та можливість легкої міграції на хмарні сервіси (RDS, Cloud SQL).
