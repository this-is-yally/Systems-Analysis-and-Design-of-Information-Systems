# Розділ 10. Система безпеки

## 10.1 Аутентифікація та авторизація

- Використовувати **паролі, збережені у хешованому вигляді** (bcrypt/scrypt/Argon2).
- Реалізувати **рольову модель** (RBAC): читач, бібліотекар, адміністратор.
- Захист від брутфорс-атак (ліміт спроб логіну, блокування).

## 10.2 Захист передачі даних

- Використовувати **HTTPS** для всіх запитів.
- Захищені кукі (HttpOnly, Secure) якщо використовується сесія.

## 10.3 Валідація та захист від атак

- Валідувати всі вхідні дані (сервер та клієнт).
- Захист від SQL-ін’єкцій (параметризовані запити).
- Захист від XSS та CSRF (токени).

## 10.4 Контроль доступу та аудит

- Логи дій (вхід, зміни даних, операції видачі/повернення).
- Перевірка прав перед виконанням критичних операцій.

## 10.5 Резервне копіювання та відновлення

- План регулярних бекапів бази даних.
- Тест відновлення даних для забезпечення безперервності роботи.
