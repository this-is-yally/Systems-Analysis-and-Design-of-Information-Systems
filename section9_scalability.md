# Розділ 9. Можливості масштабування системи

## 9.1 Масштабування веб-сервера

- **Горизонтальне масштабування**: розгортання декількох екземплярів API за балансувальником навантаження (Load Balancer).
- **Стейтлесс-сервіси**: зберігання сесій у Redis або JWT дозволяє додавати/видаляти інстанси без втрати стану.

## 9.2 Масштабування бази даних

- **Реплікація** (read replicas) для розвантаження операцій читання.
- **Шардинг / партіціювання** (partitioning) при великих обсягах даних (наприклад, за роком видач).
- **Індекси** для швидких пошуків по ISBN, автору та імені користувача.

## 9.3 Кешування

- **Кешування запитів** (Redis / Memcached) для частих операцій (каталог, популярні книги).
- **CDN** для статичних ресурсів (якщо є фронтенд).

## 9.4 Моніторинг і автошкала

- Метрики (CPU, пам’ять, кількість запитів) для автошкалювання.
- Логи та трасування для виявлення «вузьких місць».
