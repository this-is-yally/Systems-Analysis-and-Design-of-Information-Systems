# Розділ 12. Вибір методології розробки

## 12.1 Рекомендована методологія

Для розробки системи рекомендовано застосувати **Agile (Scrum)**:

- Розбити роботу на **спринти** (1–2 тижні).
- Створити **product backlog** з user stories з попередніх розділів.
- Проводити **щоденні стендапи**, **ретроспективи** та **демо**.

## 12.2 Чому Scrum?

- Дає гнучкість для змін вимог.
- Допомагає частіше отримувати зворотний зв’язок.
- Дозволяє контролювати та оцінювати прогрес через інкременти.

## 12.3 Альтернативи

- **Kanban** — підходить для безперервного потоку задач.
- **Waterfall** — не рекомендується для проектів з невизначеними вимогами.
